# Main Agent 訪談彙整

> 這是 Codex 依 persona 訪談整理的重點版。內容仍需小組確認，不是最後產品決策。

## 共同重點

- 三個 persona 都需要清楚分辨：原始資訊、整理草稿、已確認事實。
- 「AI 輔助可信度」「排序」「分數」很容易被誤讀成查核結果或行動優先序。
- 「草稿」「儲存」「候選整理」容易被誤會成正式保存、確認完成，或可以準備行動。
- v1 即使先服務資訊整理者，也不能讓回報者的「轉述、不確定」消失，也不能讓行動者誤以為可以出發。

## 最大風險

- 把未確認資訊顯示得太完整，讓人以為已經可信。
- 把來源型別誤當成可信度或官方背書。
- 把 AI 推測、整理者判斷、原文混在一起。
- 把「可整理」誤讀成「可派工」或「可行動」。

## v1 取捨

- v1 優先服務資訊整理者。
- 不做可信度排名、派工優先序或行動建議。
- 草稿要分開呈現「原文」「整理者判斷」「仍待確認」。
- 「是否親眼確認 / 是否轉述 / 是否過期」要成為畫面上的明確訊號。
- 判斷理由可以考慮用固定勾選加短文字，避免只靠自由文字。

## 不讓 AI 自動決定

- 不自動把原始資訊改成已確認。
- 不自動判斷是否可以派工或前往現場。
- 不自動補真實地點、人物、電話、數量或需求。
- 不自動判斷當事人同意、集合點安全性、資訊是否仍有效。

## 待人類確認

- 是否完全移除分數，或改成低權重的待確認提示。
- 「候選整理」是否改成更保守的文案，例如「可整理但不可行動」。
- v1 是否以「原文 / 我的判斷 / 仍待確認」作為核心畫面結構。
- 即使不做行動者流程，畫面是否仍需要明顯的「不能直接行動」訊號。

## 小隊討論後補充

- 待補：小隊採用的 AI 建議。
- 待補：小隊不採用的 AI 建議。
- 待補：小隊自己的判斷與仍不確定的地方。
